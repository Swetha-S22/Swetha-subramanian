  • When a command line is run it generates two outputs : output and error
  • output goes to :stdout and Error goes to : stderr
  • Both stdout (1) and stderr ( 2) have a specific file descriptor. Using it we can redirect the stdout and stderr to other files.
  • /dev/null : redirect standard error messages to /dev/null (to send permission denied errors).Since it is not an executable file cannot use piping | to redirect. Have to use > or >>, <or <<
